Система Управления Заказами в Службе Доставки
Это консольное приложение представляет собой симулятор системы управления заказами для службы доставки еды. Проект создан для демонстрации применения основных принципов ООП и паттернов проектирования на языке C#.

✨ Ключевые паттерны, использованные в проекте
Цель лабораторной работы — применить 5 паттернов проектирования для создания гибкой и расширяемой архитектуры.

**Строитель (Builder)**
Используется для пошагового создания сложных объектов Order. Это позволяет избежать конструктора с большим количеством параметров и делает процесс создания заказа более наглядным.
**Фабричный метод (Factory Method)**
Применяется для создания объектов различных типов доставки (StandardDelivery, ExpressDelivery). Паттерн делегирует создание экземпляров подклассам, что позволяет легко добавлять новые способы доставки в будущем.
**Состояние (State)**
Реализует логику смены статусов заказа (Готовится -> Доставляется -> Выполнен). Поведение объекта Order меняется в зависимости от его внутреннего состояния, что избавляет от громоздких if/switch конструкций.
**Стратегия (Strategy)**
Используется для инкапсуляции различных алгоритмов расчета стоимости заказа (StandardCostStrategy, DiscountCostStrategy). Это позволяет менять алгоритм расчета "на лету", не изменяя классы, которые его используют.
**Наблюдатель (Observer)**
Организует систему уведомлений. При изменении состояния заказа (Order) все подписанные наблюдатели (например, SmsNotifier) автоматически получают уведомление и реагируют на него.
